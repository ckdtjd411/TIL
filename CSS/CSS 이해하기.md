# CSS 기초

## 1. CSS 이해하기

### 1.1 CSS 소개

#### CSS와 HTML

CSS는 간단히 이야기하면 HTML(마크업 언어)를 꾸며주는 표현용 언어

마크업 언어(HTML)는 구조를 담당하고 CSS는 외적으로 꾸며주는 역할

웹 사이트들은 모두 HTML 태그를 이용하여 만들어졌고, 자주 사용되는 태그의 개수는 10여 개밖에 되지 않는다.

결국, 모든 사이트가 비슷한 HTML 태그를 사용해서 만들어졌음에도 각각 다양하고 독창적인 디자인으로 표현 가능한 이유가 CSS 덕분



### 1.2 CSS 문법과 적용

#### CSS 구문

h1 { color: yellow; font-size: 2em;}

- 선택자(selector) : 어느 요소를 꾸밀 것인지 - "h1"
- 속성(property) : 어떤 모양을 꾸밀지 - "color"
  - HTML 속성(attribute) vs CSS 속성(property)
    - Attributes는 HTML 요소의 추가적인 정보 전달하고 이름="값"
    - Property는 attribute에 대한 HTML DOM 트리 안에서의 표현
- 값(value) : 어떻게 꾸밀지 - "yellow"
- 선언(declaration) : 속성 + 값 - "color: yellow", "font-size: 2em"
- 선언부(declaration block) - "{ color: yellow; font-size:2em; }"
- 규칙(rule set) - "h1 { color: yellow; font-size:2em; }"



#### 주석

/* 내용 */



#### CSS 적용

**1) Inline**

해당 요소에 직접 스타일 속성을 이용해서 규칙들을 선언하는 방법

해당 요소에 직접 입력하기 때문에 선택자 필요X, 선언부에 내용만 스타일 속성의 값으로 입력

\<div style="color: red;">내용</div>



**2) Internal**

문서에 \<style>을 활용한 방법

\<head> 내부에 들어가며 \<style> 안에 스타일 규칙 작성

많은 페이지가 있는 경우 모든 페이지에 저마다 규칙을 선언해야 하므로 어려움 있음

\<style> div {color:red;} \</style>



**3) External**

외부 스타일 시트 파일을 이용한 방법

스타일 규칙들을 별도의 외부 파일을 만들어 넣는 방식

외부 파일의 확장자는 .css

\<link rel="stylesheet" href="css/style.css">



**4)  Import**

스타일 시트 내에서 다른 스타일 시트 파일을 불러오는 방식

\<style> 내부 상단이나 외부 스타일 시트 파일 상단에 선언

성능상 좋지 않아서 거의 사용하지 않음



### 1.3 선택자

#### 요소 선택자

선택자 중에 기본이 되는 선택자이며, 태그 선택자라고도 한다

선택자 부분에 태그 이름이 들어가며 문서 내에 선택자에 해당하는 모든 요소에 스타일 규칙이 적용



##### 그룹화

선택자는 쉼표를 이용해서 그룹화 가능

h1, h2, h3, h4, h5, h6 { color: yellow; }

*(별표, asterisk) : 문서 내에 모든 요소 선택

(성능에 좋지 않으므로 사용을 지양)



#### class 선택자

요소에 구애받지 않고 스타일 규칙을 적용할 수 있는 일반적인 방법

글로벌 속성이므로 어느 태그에서도 사용 가능

클래스 선택자를 쓸 때는, 맨 앞에 .(마침표)를 찍어야 함

공백으로 구분하여 여러 개의 class 값을 사용



#### id 선택자

선택자를 쓸 때는 #(해시) 기호 사용

<u>class 선택자와의 차이점</u>

1. 기호가 아닌 # 기호 사용
2. 태그의 class 속성이 아닌 id 속성을 참조
3. 문서 내에 유일한 요소에 사용
4. 구체성



#### 속성 선택자

##### 단순 속성으로 선택

속성 선택자는 대괄호를 이용해서 선언하며 대괄호 안에 속성 이름이 들어간다.

요소에 해당 이름의 속성이 있다면 해당 사항이 적용

`p[class] {color: silver}`



##### 정확한 속성값으로 선택

속성의 값으로 요소를 선택

선택자는 대괄호 안에 속성 이름과 속성값을 입력

`p[class="foo" { color: silver; }`



##### 부분 속성값으로 선택

속성 이름과 속성값 사이에 사용되는 기호에 따라 동작이 다름

\[class~="bar"] : 공백으로 구분한 "bar" 단어가 포함되는 요소

\[class^="bar"] : "bar"로 시작하는 요소

\[class$="bar"] : "bar"로 끝나는 요소

\[class*="bar"] : "bar" 문자가 포함되는 요소



#### 문서 구조 관련 선택자

문맥이나 요소의 구조를 기반으로 하여 선택자를 조합하는 것을 "조합자" 또는 "결정자"라고 부른다

조합자를 이용하면 문서 구조를 이용해 좀 더 유연하게 요소를 선택하고 스타일 적용 가능



##### 부모와 자식 관계

부모 요소는 그 요소를 포함하는 가장 가까운 상위 요소, 그 요소의 부모 요소는 단 하나

자식 요소는 부모 요소의 반대로 자식 요소는 여러 개일 수도 있다

##### 조상과 자손

부모와 자식의 관계를 포함한 확장된 관계

조상 요소는 그 요소를 포함하는 모든 요소로, 부모 요소를 포함하여 여러 개일 수 있다

자손 요소는 그 반대로, 그 요소가 포함하고 있는 모든 요소가 자손 요소

##### 형제

같은 부모를 가지고 있는 요소들



**자손 선택자**

선택자 사이에 아무 기호없이 그냥 공백으로 구분

div span { color: red; }

**자식 선택자**

선택자 사이에 닫는 꺽쇠 기호(>)

div > h1 { color: red; }

**인접 형제 선택자**

선택자 사이에 + 기호

형제 관계이면서 바로 뒤에 인접해 있는 요소를 선택하는 선택자

body > div table + ul { .... } 



#### 가상 선택자

##### 가상 클래스

미리 정의해놓은 상황에 적용되도록 약속된 보이지 않는 클래스

약속된 상황이 되면 브라우저 스스로 클래스를 적용

```css
:pseudo-class {
    property: value;
}
```

가상 클래스는 :(콜론) 기호를 써서 나타냄



##### 문서 구조와 관련된 가상 클래스

- :first-child : 첫 번째 자식 요소 선택
- :last-child : 마지막 자식 요소 선택



##### 링크 관련된 가상 클래스

- :link : 하이퍼링크이면서 아직 방문하지 않은 앵커
- :visited : 이미 방문한 하이퍼링크를 의미



##### **사용자 동작과 관련된 가상 클래스**

- :focus: 현재 입력 초점을 가진 요소에 적용
- :hover: 마우스 포인터가 있는 요소에 적용
- :active: 사용자 입력으로 활성화된 요소에 적용



##### 가상 요소

미리 정의해놓은 위치에 삽입이 되도록 약속되어 있는 보이지 않는 요소

선언 방법은 가상 클래스와 같게 콜론을 사용 (CSS3부터는 ::(더블 콜론) 기호를 사용)

- :before : 가장 앞에 요소를 삽입
- :after : 가장 뒤에 요소를 삽입
- :first-line : 요소의 첫 번째 줄에 있는 텍스트
- :first-letter : 블록 레벨 요소의 첫 번째 문자



### 구체성

#### 구체성(명시도)

어떤 스타일이 적용?

```css
h1 { color: red; }
body h1 { color: green; }
```

h1은 green이 적용 -> 구체성과 연관



구체성은 선택자를 얼마나 명시적으로 선언했느냐를 수치화 한 것

구체성의 값이 클수록 우선으로 적용

- 0, 1, 0, 0 : 선택자에 있는 모든 id 속성값
-  0, 0, 1, 0 : 선택자에 있는 모든 class 속성값, 기타 속성, 가상 클래스
-  0, 0, 0, 1 : 선택자에 있는 모든 요소, 가상 요소
-  전체 선택자는 0, 0, 0, 0을 가진다.
-  조합자는 구체성에 영향을 주지 않는다. (>, + 등)



#### 인라인 스타일

인라인 스타일의 구체성 값은 1, 0, 0, 0 이며 규칙들 중 가장 큰 구체성을 갖는다



#### important

important 키워드는 별도의 구체성 값은 없지만, 모든 구체성을 무시하고 우선권을 갖는다

속성값 뒤 한 칸 공백을 주고 느낌표 기호와 함께 쓴다

`p#page { color: red !important; }`



### 상속

어떤 스타일 규칙이 특정 요소뿐만 아니라 그 자손 요소까지 적용되는 것

#### 상속되는 속성
```markdown
CSS
	h1 { color: gray; }
HTML
	<h1>Hello, <em>CSS</em></h1>
```

\<em>은 부모인 <h1>의 color: gray를 상속받는다.

 박스 모델 속성들은 상속되지 않음 (margin, padding, background, border 등)



#### 상속된 값의 구체성은?

상속된 속성은 아무런 구체성을 가지지 못한다.



### 캐스케이딩

cascading은 스타일 규칙들이 어떠한 기준으로 요소에 적용되는지를 정한 규칙

#### cascading 규칙

1. 중요도&출처

   important로 선언된 모든 규칙은 그렇지 않은 규칙보다 우선

   CSS 출처는 제작자, 사용자, 사용자 에이전트(User Agent)로 구분

   <u>우선 순위</u>

   ​	5) 사용자 에이전트 스타일 : 브라우저에서 기본적으로 제공하는 스타일

   ​	4) 사용자 스타일 : 웹 페이지를 방문하는 일반 사용자들이 작성한 CSS

   ​	3) 제작자 스타일 : 사이트를 실제 제작하는 개발자가 작성한 CSS (대부분의 경우)

   ​	2) 제작자 !important 스타일

   ​	1) 사용자 !important 스타일

3. 선언 순서
